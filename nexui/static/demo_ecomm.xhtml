What is it?

You are looking at the demo fronted to SCAP as defined in nexo-FAST. It turns out that it's general design fits nicely into different use cases. Nexo can be configured in multitude ways one of which is one-shot (without asking any additional questions) transaction with manually entered card data. Which is basically a transaction in e-commerce paradigm. This demo showcases that.

Is it all nexo implementation can do?

It's far from the only uses of the this implementation. Arguably it's most suited to be embedded into a terminal (PoS) software, but given modular design it can be used in many different scenarios like this one.

How does it work?

All of the business logic is defined by nexo-FAST and is implemented in the back-end, which controls this small UI app.

Where is 3D-Secure?

3D Secure is irrelevant for this demo, it should be implemented in production by redirecting user to the appropriate page.
